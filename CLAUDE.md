# enot - an earthy color scheme robust to dichromacy

The enot theme (vim, lightline, sublime, vscode, bat, wezterm, mc, ranger, nimble, telegram, ohmyzsh, site). All colors are
generated by the optimizer; never hand-pick colors or edit generated
artifacts by hand. Module docstrings are the authoritative descriptions;
this file holds the map and the rules only.

## Pipeline

Layout: the modules and the optimizer state enot.json live in pipeline/,
port sources in ports/<app>/, rendered artifacts in dist/<app>/.

- palette.py - shared constants, LCh -> sRGB math, the flint base and enot_variant() assembling the theme from enot.json;
- optimize.py -> enot.json: 7 accents + 16 ANSI per theme, max-min pairwise CIEDE2000 across normal vision, protanopia and deuteranopia, then an aesthetic pass toward gruvbox anchors;
- resolve.py -> colors.json: every role at three depths (hex, xterm-256, ANSI slot); 256 is solved by the same optimizer over xterm-grid candidates, not by nearest-color (which merges roles down to dE00 0); the single source of colors for renderers;
- check.py - the single regression: metric thresholds per depth + artifact conformance derived from ports/*/port.json, so a new port is checked without touching check.py; cvd.py holds the simulation matrices and dE00 only;
- build.py - the port engine: renders every port under ports/ from colors.json (no color math) into dist/<app>/ at the paths declared in the manifest and writes ports/registry.json (the coverage manifest); a port is ports/<app>/port.json of kind template (a ${var} file, no Python), module (render.py) or static (copied verbatim); a new simple app is one template + one manifest;
- assets.py -> docs/assets/*.svg: the README design reference; editor and palette sheets are hex-checked, cvd-{dark,light}.svg are exempt (foreign palette and simulation outputs);
- sitedata.py -> build/site/: the pipeline's contract with the site - site.json bundle (precomputed vision simulations, metrics, gates, numbers, port registry), llms*.txt and downloads, all derived from the registry; downloads are app-scoped (/<app>/<file> - identical basenames across ports must not collide); the site never recomputes colors;
- make build - full pipeline, the CI gate; make deploy - build + sitedata, then sync the bundle into the Astro site and push. CI fails on any check violation or when artifacts are not regenerated from the spec.

## Invariants

- changing colors = changing constraints or anchors in optimize.py (truecolor) or tolerances in resolve.py (256), then recompute; after any change run make all, check must pass;
- dichromacy models: Vienot 1999 (protan/deutan), Brettel 1997 (tritan); Machado 2009 is only valid for anomalous trichromacy;
- tritanopia is not part of hard constraints (project decision); it is simulated only for the site's vision switcher;
- contrast of text roles to background >= 4.5:1 at all depths; distance from fg/comment >= dE00 10 (bright ANSI and 256 accents - 6.5, stricter is physically unattainable); red C* >= 22 (21.9 after hex roundtrip, threshold 21.5);
- ANSI: normal and bright lightness tiers do not overlap; in the light theme bright is darker than normal (like gruvbox light);
- 256 depth: dark accents >= 7.0, light >= 2.0 - the physical limit of the xterm grid at 4.5:1 contrast (the aqua-blue pair); the full guarantee holds only in truecolor;
- vim scheme attributes are always explicit (gui=NONE), otherwise StatusLine/IncSearch inherit reverse.

## Site

- site/ - a symlink to the sibling checkout of git@github.com:enot-theme/enot-theme.github.io.git, an Astro project (not build output), served at the organization root https://enot-theme.github.io/ ; Pages deploys from GitHub Actions on push, directory-style URLs (/vim/, /apps/);
- the site owns presentation and per-app words (src/data/install.ts); the pipeline owns numbers and names (src/data/site.json, synced by make deploy); install pages, cards, downloads and the coverage matrix generate from the registry, so a new port needs zero site edits; never hand-edit src/data/site.json or public/ scheme files;
- wezterm install snippets - config_builder style, no top-level return in examples;
- analytics: GoatCounter, code in site/src/data/config.ts (register the code to receive data; '' disables).

## Language and commits

- the organization language and the language of all artifacts without exception is English: sources, comments, palette names, docs, site, commits;
- commits: semantic, one line, up to 80 chars, English, DCO (git commit -s), no Co-Authored-By.

## Documentation

- docs/enot.en.md is the single method document; when the method or numbers change, update it, then make sitedata regenerates llms-full.txt and make deploy publishes it;
- key numbers: min dE00 13.3 dark / 11.1 light (accents) and 7.2 (ANSI) in truecolor, 8.3/3.8 (256 accents, dark/light); the site takes numbers from the synced bundle automatically; in write-ups verify against metrics.
